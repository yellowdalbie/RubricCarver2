# 루브릭 템플릿 (RubricCarver 2.0)

> 이 파일은 `rubrics/v00_rubric.md` 작성 시 참조하는 형식 가이드입니다.
> 실제 루브릭은 문항에 맞게 내용을 채워 `rubrics/v00_rubric.md`에 저장하고,
> 실험 시작 전 `rubrics/master_rubric.md`로 복사합니다.

---

## 구조 제약 (engine/main.py PG-1 검증 조건)

1. "사전 검사 조건" 또는 "Pre-check Gate" 문자열이 포함된 섹션이 있어야 함
2. C1-1, C1-2, C2-1, C2-2, C2-3, C3-1, C3-2, C3-3, C3-4, C3-5 코드가 모두 있어야 함
3. `[개념]` 태그 기준이 3개 이상이어야 함 (분석관 검증 조건)

## 기준 작성 지침

### [절차] 기준 작성 요령
- "~을 수식으로 표현했는가"
- "~공식/정리를 기호로 명시했는가"
- 학생 답안에서 *있다/없다*로 판별 가능한 형태

### [개념] 기준 작성 요령
- "~의 의미/이유를 서술했는가"
- "~관계가 논리적으로 드러나는가" (수식 없이 서술로도 가능)
- 학생이 *이해하고 있음을 드러내는 표현*이 있는가로 판별

### 기준 코드 배정 원칙
- C1-x: 문항의 첫 번째 풀이 단계 (기초 변환/설정)
- C2-x: 두 번째 단계 (핵심 조건 변환/계산)
- C3-x: 세 번째 단계 (연립풀이/최종 답)
- 각 단계에 절차 기준과 개념 기준이 균형 있게 배분되도록 설계

### 초기 비율 권장
- 초기 루브릭: `[절차]` 6개 이하, `[개념]` 4개 이상
- 이후 진화 방향 관찰이 실험의 핵심 관심사
