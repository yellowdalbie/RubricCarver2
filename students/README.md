# 학생 답안 디렉토리

## 파일 명명 규칙

```
student_{ID}.md
```

## 학생 구성 (16명)

| ID | 성취수준 | 설계 목적 |
|----|----------|-----------|
| A1 | A | 최우수 — 수식+서술 모두 완벽 |
| A2 | A | 최우수 — 수식 없이 서술만으로도 A 수준 표현 (경계 케이스 후보) |
| B1 | B | 우수 — 대부분 정확, 일부 단계 누락 |
| B2 | B | 우수 — 개념 이해 드러나나 수식 부정확 (T1/T3 불일치 유발 목적) |
| C1 | C | 보통 — 핵심 개념 파악, 수식 불완전 |
| C2 | C | 보통 — 수식 있으나 논리 흐름 불완전 |
| C3 | C | 보통 — 특정 기준에서 T1↔T3 불일치 유발 설계 |
| D1 | D | 미흡 — 일부 단계만 수행, 나머지 누락 |
| D2 | D | 미흡 — 오개념 포함 |
| D3 | D | 미흡 — 부분적 이해만 표현 |
| E1 | E | 최하 — 풀이 있으나 전혀 올바르지 않음 |
| E2 | E | 최하 — 매우 단편적 서술 |
| F1 | F | 답안 없음 또는 "모르겠습니다" 수준 |
| F2 | F | adversarial injection 포함 답안 (PG-1 robustness 테스트) |
| I1 | I | 오프토픽 (문항과 무관한 답안) |
| I2 | I | 판독 불가 / 의미 없는 텍스트 |

## 설계 원칙

1. **T1/T2 vs T3/T4 불일치 유발**: B2, C3 학생은 "수식 없이 개념을 서술한 경우"로 설계하여  
   절차중심 교사(T1/T2)와 개념중심 교사(T3/T4) 간 불일치를 의도적으로 유발해야 한다.

2. **T1/T3 vs T2/T4 불일치 유발**: 일부 학생은 앞 단계 오류가 뒤 단계에 영향을 미치는 구조로 작성하여  
   독립판단(T1/T3)과 연계판단(T2/T4) 간 불일치를 유발해야 한다.

3. **경계 케이스**: 루브릭 확정 후 `experiment_design/expected_scores_matrix.json`의  
   `boundary_cases` 필드와 `engine/main.py`의 `BOUNDARY_CASES` 변수를 동기화한다.

4. **adversarial 답안 (F2)**: prompt injection 시도를 포함하여  
   PG-1 게이트 robustness를 테스트한다.
